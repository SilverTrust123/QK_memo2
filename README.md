1.用了sliding window 可以加在報告裡面
抑制感測器噪訊
能當作低通濾波器過濾毛刺
提升SNR、增強數據穩定性

滑動窗口確保了警報是基於「持續性的異常」而非「瞬時的跳變」。只有當窗口內的平均值持續超過閾值，系統才會觸發報警。

優點： 這大大提升了自動化系統的 決策魯棒性 (Robustness)。

2.也算了 rps 也可以加在報告裡面

下面是計算公式
rps = rpst / sec

負載總量估計
單片機(esp8266 wifi) 5台 = 5rps
cam 估計2台 = 2RPS
前端輪詢 6 / 1 = 12RPS
不定時(緊急) 1RPS

sum = 20 - 25 RPS
依據spring-boot 的thread pools 規格得知最大是200
